Case 03 · Windows Logs

Suspicious activity timeline with MITRE mapping.

Problem

Single Windows events are often weak alone. The real signal appears when user, host, process, and network activity connect.

Solution

Review Windows/Sysmon-style events, order by time, identify suspicious process behavior, and map to MITRE tactics.

Workflow

Noise → triage → evidence → verdict.

My L1 process is intentionally simple: understand the alert, validate with evidence, write clean notes, and escalate only with a reason.

  1. 01Alert intake

    User, host, source, timestamp, rule logic, and baseline.

  2. 02Evidence review

    Logs, reputation, headers, event IDs, process and timeline indicators.

  3. 03Verdict

    Benign, suspicious, needs escalation, or needs monitoring.

  4. 04Escalation note

    Severity, reason, evidence, MITRE mapping, and next action.
